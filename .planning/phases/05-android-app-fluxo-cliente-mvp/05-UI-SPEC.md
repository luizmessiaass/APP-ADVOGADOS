# Phase 5 — UI Design Contract
## Android App: Fluxo Cliente (MVP)

> Visual and interaction contract for the `:app-cliente` module.
> Generated by gsd-ui-researcher. Verified by gsd-ui-checker.
>
> Audience: non-lawyers (leigos) in Brazil. Plain language and accessibility are first-class requirements throughout.
> All locked decisions are sourced from `05-CONTEXT.md` (D-01 through D-13) and `05-RESEARCH.md`.
> APP-14 ("Explicação gerada por IA" disclaimer) is REMOVED by user decision (D-06). Executor MUST NOT implement any AI disclaimer anywhere.

| Property | Value | Source |
|----------|-------|--------|

## Spacing Scale

All spacing values are multiples of 4. Translate to `dp` in Kotlin (e.g., `16.dp`).

| Token | Value | Kotlin | Usage |
|-------|-------|--------|-------|
| xs | 4dp | `4.dp` | Icon gaps, divider padding, chip internal padding |
| sm | 8dp | `8.dp` | Compact element spacing, icon-to-label gap |
| md | 16dp | `16.dp` | Default horizontal screen padding, card internal padding |
| lg | 24dp | `24.dp` | Section vertical spacing, between cards |
| xl | 32dp | `32.dp` | Layout gaps, between major screen sections |
| 2xl | 48dp | `48.dp` | Major section breaks (e.g., below onboarding illustration) |
| 3xl | 64dp | `64.dp` | Page-level top/bottom padding on LGPD consent screen |

**Exceptions:**

| Exception | Value | Reason |
|-----------|-------|--------|
| Minimum touch target (all tappable elements) | 48dp × 48dp | Android Material accessibility requirement; minimum for leigo users |
| "Falar com meu advogado" floating button | 56dp height | Material3 ExtendedFloatingActionButton standard height |
| Timeline month/year sticky header height | 36dp | Compact sticky header; uses `labelLarge` text at 12dp vertical padding top+bottom |
| "Ver mais" TextButton tap area | 48dp minimum height | Touch target rule — even though text is small, tap area must be 48dp |

## Typography

All roles map directly to Material3 `Typography` slots in `:core-ui/Type.kt`. Roboto is the system default on Android — no additional font dependency needed.

| Role | Material3 Slot | Size | Weight | Line Height | Usage |
|------|---------------|------|--------|-------------|-------|
| Display | `headlineLarge` | 28sp | SemiBold (600) | 1.2 (34sp) | Onboarding screen main headline only |
| Heading | `titleLarge` | 20sp | SemiBold (600) | 1.2 (24sp) | Screen titles (toolbar), section headers ("Movimentações", "Dados do Processo") |
| Body | `bodyLarge` | 16sp | Regular (400) | 1.5 (24sp) | Card body text, translated `explicacao` text, LGPD policy paragraphs, onboarding subtitles |
| Label | `labelLarge` | 14sp | Regular (400) | 1.4 (20sp) | Timeline month/year headers, metadata (tribunal, vara labels), "sincronizado há X horas", "Ver mais" buttons, checkbox label |

**Rules:**
- Maximum 2 weights in use: Regular (400) and SemiBold (600). Do not use Medium (500) or Bold (700).
- Maximum 4 type sizes: 28sp, 20sp, 16sp, 14sp. Do not introduce additional sizes.
- Letter spacing: use Material3 defaults (0sp for headings, 0.5sp for body/label).
- Do not use italic anywhere — the audience includes low-literacy users for whom italics reduce readability.

## Color

The existing scaffold uses Material3 default purple tones (Purple40 = `#6650A4`). Phase 5 replaces these with a trust-oriented palette appropriate for legal services. All values go into `:core-ui/Color.kt` and are applied via `PortalJuridicoTheme`.

Dynamic color (`dynamicColor = true`) introduced in the scaffold is **disabled** for `:app-cliente`. Reason: white-label white-label tenants require predictable brand colors, not system-driven dynamic colors that change per device wallpaper.

| Role | Light Value | Dark Value | Usage | Distribution |
|------|-------------|------------|-------|-------------|
| Dominant — Background | `#FAFAFA` (near-white) | `#121212` | Screen background, Scaffold background | 60% |
| Dominant — Surface | `#FFFFFF` | `#1E1E1E` | Cards (status card, próxima data card, movimentação cards), bottom sheets | 60% (shared) |
| Secondary — Surface Variant | `#F0F4F8` (light blue-grey) | `#2A2A2A` | LGPD consent screen background, onboarding page background, section separators | 30% |
| Accent — Primary | `#1A56DB` (deep blue) | `#4D9BFF` | CTA buttons ("Entrar", "Aceitar", "Começar", "Próximo"), active pager dot, checkbox checked state, "Falar com meu advogado" FAB | 10% |
| Accent — Primary Container | `#DBEAFE` (light blue) | `#1E3A5F` | Status card background (current process status — IA-translated), próxima data card highlight | 10% (shared) |
| Destructive | `#DC2626` (red) | `#F87171` | LGPD rejection path only — "Sair" button on rejection confirmation dialog | Rare |
| On-Primary | `#FFFFFF` | `#FFFFFF` | Text/icons on accent (#1A56DB) buttons | — |
| On-Surface | `#1C1B1F` | `#E6E1E5` | All body text on white/light cards | — |
| Outline | `#C4C4C4` | `#4A4A4A` | Card borders, dividers, text field outlines | — |

**Accent reserved for:**
1. Primary CTA buttons only: "Entrar" (login), "Próximo" (onboarding), "Começar" (last onboarding page), "Aceitar" (LGPD)
2. "Falar com meu advogado" ExtendedFAB background
3. Active pager indicator dot on onboarding
4. Checkbox checked state (LGPD consent)
5. Status card and próxima data card backgrounds (Primary Container, not Primary)

**Accent is NOT used for:** body text, timeline items, labels, disabled states, data cadastral section, month/year headers.

**White-label override point:** The `primary` color (`#1A56DB`) is the single brand color that future per-office flavors may override via flavor-specific `Color.kt`. This is the only color the white-label mechanism touches. All other colors remain fixed.

## Interaction and State Contracts

### Loading States

Every screen that fetches data must implement these states in the ViewModel `UiState` sealed class:

| State | Visual | Component |
|-------|--------|-----------|
| Loading | Skeleton cards with animated shimmer (alpha 0.3 → 0.8 → 0.3 loop) | `Surface` with `animateFloatAsState` |
| Success | Actual content | Screen composable |
| Error | Error message + retry button | Centered `Column` |
| Empty | Empty state illustration + copy | Centered `Column` |

### Navigation Flow

```
Login
  └─► (first open, !hasSeenOnboarding) Onboarding (4 pages)
        └─► (!hasAcceptedLgpd) LGPD Consent Gate
              └─► Process List
                    └─► Process Detail
                          └─► (back) Process List
  └─► (returning user, hasSeenOnboarding && hasAcceptedLgpd) Process List

LGPD Reject ─► (logout) Login
```

Start destination is determined before NavHost renders (SplashViewModel reads DataStore flags; NavHost only mounts after all flags are non-null — avoids screen flicker; see RESEARCH.md Pitfall 4).

### Back Navigation Rules

| Screen | Back behavior |
|--------|---------------|
| Login | No back (root destination) |
| Onboarding page 1 | BackHandler no-op (cannot exit onboarding) |
| Onboarding pages 2–4 | Back navigates to previous page (within pager) |
| LGPD Consent | Back triggers the "Recusar" rejection flow (AlertDialog), not a silent navigation |
| Process List | Back exits the app (`finish()` on Activity) |
| Process Detail | Back navigates to Process List |

### Animation

| Transition | Contract |
|------------|----------|
| Onboarding page swipe | Default `HorizontalPager` swipe animation (no custom interpolator needed) |
| Dados cadastrais expand/collapse | `AnimatedVisibility` with `expandVertically() + fadeIn()` enter; `shrinkVertically() + fadeOut()` exit; duration: 200ms |
| "Ver mais" text expansion | `animateContentSize()` modifier; default spring animation |
| Screen-to-screen navigation | Navigation Compose default slide transitions (no custom animator in Phase 5) |
| Loading skeleton shimmer | `animateFloatAsState` alpha loop, 800ms cycle |

## Accessibility Contract

The audience includes low-literacy users and older Brazilians. These rules are mandatory, not optional.

| Rule | Implementation |
|------|----------------|
| Minimum touch target 48dp × 48dp | Apply `Modifier.defaultMinSize(minWidth = 48.dp, minHeight = 48.dp)` to all clickable composables that are smaller than 48dp visually |
| Content descriptions on all icons | All `Icon()` calls must have `contentDescription` with meaningful Portuguese text; never `contentDescription = null` for actionable icons |
| Color contrast — body text on white | `#1C1B1F` on `#FFFFFF` = 18.1:1 contrast ratio — passes WCAG AAA |
| Color contrast — body text on Primary Container | `#1C1B1F` on `#DBEAFE` = 12.1:1 — passes WCAG AAA |
| Color contrast — accent text links | `#1A56DB` on `#FFFFFF` = 7.2:1 — passes WCAG AA (large text) + AA (normal text) |
| Semantic roles | All `Button`, `Checkbox`, `Card(onClick)` composables use Material3 semantics automatically; no custom `Role` needed unless using `Modifier.clickable` on a `Box` |
| System font scaling | All `sp` values (not `dp`) for text — respects system accessibility font size setting |
| Focus order | Compose default focus order follows composition order — ensure `LazyColumn` item order matches reading order |
