    - skipAuth: true (no JWT — server-to-server)
    - Validate X-Webhook-Secret header against env.BILLING_WEBHOOK_SECRET (constant-time comparison via timingSafeEqual)
    - Invalid/missing secret → 401 { error: 'unauthorized', code: 'INVALID_WEBHOOK_SECRET' }
    - Validate body with Zod: { event: z.enum(['payment.failed','payment.succeeded']), tenant_id: z.string().uuid(), event_id: z.string().min(1), occurred_at: z.string(), provider: z.string().optional(), metadata: z.record(z.unknown()).optional() }
    - Invalid body → 400 { error: 'invalid_payload', code: 'VALIDATION_ERROR' }
    - Idempotent insert: INSERT INTO billing_events (...) ON CONFLICT (event_id) DO NOTHING RETURNING id
    - If RETURNING is empty (duplicate) → return 200 { status: 'already_processed' }
    - payment.failed: UPDATE escritorios SET status='grace', grace_period_started_at=now(), updated_at=now() WHERE id=tenant_id AND status NOT IN ('grace','read_only','suspended') — only if not already in grace
    - payment.failed: also insert billing_events row with event='payment.failed'
    - payment.succeeded: call resolveGracePeriod logic: UPDATE escritorios SET status='active', grace_period_started_at=NULL, grace_banner=false, updated_at=now() WHERE id=tenant_id
    - Always invalidate Redis cache: invalidateTenantStatusCache(redis, tenant_id)
    - Return 200 { status: 'processed' }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| External payment provider → POST /api/webhooks/billing | Unauthenticated server-to-server call; only X-Webhook-Secret protects it |
| super_admin JWT → admin endpoints | JWT claim can be tampered if signing key is leaked; JWKS validation in authPlugin is the primary guard |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-7-08 | Spoofing | POST /api/webhooks/billing | mitigate | `crypto.timingSafeEqual` comparison of X-Webhook-Secret against BILLING_WEBHOOK_SECRET env var — prevents timing attacks; rejects missing or wrong secret with 401 |
| T-7-09 | Tampering | Replay of old payment.succeeded | mitigate | billing_events UNIQUE(event_id) — INSERT ON CONFLICT DO NOTHING prevents replay at DB level |
| T-7-10 | Elevation of Privilege | GET /api/admin/tenants cross-tenant | mitigate | supabaseAdmin (service role) used deliberately with explicit super_admin role check in handler — no RLS bypass vulnerability because the check is intentional |
| T-7-11 | Tampering | PATCH /api/admin/tenants/:id/status with invalid status | mitigate | Zod enum validation on body.status rejects values not in allowed set before DB write |
| T-7-12 | Information Disclosure | admin tenant list exposes all tenant PII | accept | Only super_admin (authenticated + role checked) can access this endpoint; not exposed publicly |
</threat_model>
