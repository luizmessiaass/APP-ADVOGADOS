**1. Criar BatteryOptimizationScreen.kt:**

```kotlin
@Composable
fun BatteryOptimizationScreen(
    onSkip: () -> Unit,       // D-06: pular é permitido
    onContinue: () -> Unit,   // após configurar (volta do settings)
    modifier: Modifier = Modifier
) {
    val context = LocalContext.current
    val packageName = context.packageName
    val manufacturer = Build.MANUFACTURER.lowercase(Locale.getDefault())

    val oemInstruction = when {
        "xiaomi" in manufacturer || "redmi" in manufacturer || "poco" in manufacturer ->
            "Configurações → Apps → Gerenciar apps → Portal Jurídico → Bateria → Sem restrição"
        "samsung" in manufacturer ->
            "Configurações → Bateria → Limites de uso da bateria → Portal Jurídico → Sem restrição"
        "motorola" in manufacturer ->
            "Configurações → Apps → Portal Jurídico → Bateria → Sem restrição no uso da bateria"
        else ->
            "Configurações → Apps → Portal Jurídico → Bateria → Sem restrição"
    }

    Column(
        modifier = modifier.fillMaxSize().padding(24.dp),
        horizontalAlignment = Alignment.CenterHorizontally,
        verticalArrangement = Arrangement.SpaceBetween
    ) {
        Column(horizontalAlignment = Alignment.CenterHorizontally) {
            // Ícone de bateria — usar ícone Material existente
            Icon(
                imageVector = Icons.Default.BatteryAlert,
                contentDescription = null,
                modifier = Modifier.size(80.dp),
                tint = MaterialTheme.colorScheme.primary
            )
            Spacer(Modifier.height(24.dp))
            Text(
                text = "Notificações confiáveis",
                style = MaterialTheme.typography.headlineSmall,
                textAlign = TextAlign.Center
            )
            Spacer(Modifier.height(12.dp))
            Text(
                text = "Para receber avisos dos seus processos mesmo quando o app estiver fechado, desative a otimização de bateria.",
                style = MaterialTheme.typography.bodyMedium,
                textAlign = TextAlign.Center,
                color = MaterialTheme.colorScheme.onSurfaceVariant
            )
            Spacer(Modifier.height(16.dp))
            Card(modifier = Modifier.fillMaxWidth()) {
                Text(
                    text = oemInstruction,
                    modifier = Modifier.padding(16.dp),
                    style = MaterialTheme.typography.bodySmall,
                    color = MaterialTheme.colorScheme.onSurfaceVariant
                )
            }
        }

        Column {
            Button(
                onClick = {
                    val intent = Intent(Settings.ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS).apply {
                        data = Uri.parse("package:$packageName")
                    }
                    context.startActivity(intent)
                    onContinue()
                },
                modifier = Modifier.fillMaxWidth()
            ) {
                Text("Configurar agora")
            }
            Spacer(Modifier.height(8.dp))
            TextButton(
                onClick = onSkip,
                modifier = Modifier.fillMaxWidth()
            ) {
                Text("Pular")
            }
        }
    }
}
```

**Nota sobre intents OEM específicos:** A pesquisa indica que intents específicos de MIUI (`miui.intent.action.APP_PERM_EDITOR`) são instáveis entre versões de firmware. Usar `ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` como chamada principal (funciona em todos os OEMs) + instrução textual específica do OEM. Não tentar abrir configurações OEM-específicas via Intent diferente.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Build.MANUFACTURER → instrução OEM | Dado do sistema — confiável mas pode ter valores inesperados |
| ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS → Settings | Intent padrão do Android — sem superfície de ataque adicional |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-06-20 | Tampering | Intent OEM-específica com extras manipulados | mitigate | Usar APENAS `ACTION_REQUEST_IGNORE_BATTERY_OPTIMIZATIONS` padrão com `data = Uri.parse("package:$packageName")` — packageName é do próprio app, não input do usuário |
| T-06-21 | Information Disclosure | Build.MANUFACTURER em logs de crash | accept | Dado de fabricante é informação de hardware não-sensível; nenhum PII envolvido |
| T-06-22 | Denial of Service | POST_NOTIFICATIONS permission launcher crashando em Android < 13 | mitigate | Guard explícito `if (Build.VERSION.SDK_INT >= Build.VERSION_CODES.TIRAMISU)` antes de lançar o request — noop em API < 33 (D-07) |
</threat_model>
