## Self-Check: PASSED

## Summary

Converted `app-escritorio` from an Android application module to a library module and wired a
single unified entry point in `app-cliente` that routes users to the correct navigation graph
based on their JWT role. One APK now serves both clientes and advogados/admins.

## What Was Built

- `app-escritorio/build.gradle.kts` — changed plugin from `android.application` to `android.library`, removed `applicationId`
- `app-escritorio/src/main/AndroidManifest.xml` — replaced with minimal library manifest (no activity/application entries)
- Deleted `app-escritorio/EscritorioApp.kt` — library modules cannot have @HiltAndroidApp
- Deleted `app-escritorio/MainActivity.kt` — library modules have no entry point
- Deleted `app-escritorio/di/AppNetworkModule.kt` — NetworkConfig now provided by app-cliente
- `app-cliente/build.gradle.kts` — added `implementation(project(":app-escritorio"))`, added `API_BASE_URL` buildConfigField for debug and release
- `app-cliente/di/NetworkModule.kt` — real Hilt module providing `NetworkConfig` via `BuildConfig.API_BASE_URL`
- `app-cliente/features/auth/LoginViewModel.kt` — `LoginUiState.Success` now carries role, removed `role != "cliente"` restriction, saves `ROLE_KEY` to DataStore
- `app-cliente/features/auth/SplashViewModel.kt` — added `ROLE_KEY`, `userRole: StateFlow<String?>`, routes `role in ADMIN_ROLES` to `Routes.ADMIN_HOME`
- `app-cliente/navigation/Routes.kt` — added `const val ADMIN_HOME = "admin_home"`
- `app-cliente/MainActivity.kt` — injects `AuthRepository`, mounts `EscritorioNavGraph` for admin roles, `ClienteNavGraph` for cliente role
- `app-cliente/features/auth/LoginScreen.kt` — navigates to `Routes.ADMIN_HOME` for admin roles after login
